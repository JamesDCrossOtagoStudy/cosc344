Summary

We implement the transaction simulation which involve 3 entity:

book, customer, employee. It further related to bookstore, transactions and book_tran.

All form input related to foreign key, we use selection list to reference this relationship.

We also use Ajax and jQuery to improve the operation experience.

Because of the limited time, we can display the bookstore, book, employee and customer table.

And can only edit bookstore and employee information.

The transactions simulation part, we simulated the process of one customer purchase some book and checkout.

Bookstore Operation

In this bookstore section, we can display the bookstore table, edit each book store by clicking the primary which is a link.

After that, we can edit the info and save. We can also create an new bookstore.

Employee Operation

In this employee section, we can do similar operation like we did in bookstore section.

We can create an new employee, edit one existing employee by click the primary key link.And we can also create a new employee.

Customer Operation

In this section, we only display the customer table.

Book Operation

In this section, we only display the book table.

Shopping Process Simulation

During this simulation process:

We must select the employee who is responsible for this purchasing operation.

select the customer who is buying the book,

select the book item and add them into shopping list,

finally, press the checkout button to finish the transaction.

Because, we don't use session, so this purchasing process only allow a one customer at one time.
